支付宝的h5页面支付流程简单描述如下：

1. 用户点击前端的支付按钮，向后端发送请求。
2. 后端根据前端传递的订单参数（商户订单号，订单名，订单描述，用户信息等），拼接成一个pay_url，这个链接会让你跳转到支付宝的支付页。
3. 前端获取到pay_url后，将当前页面重定向至支付宝的支付页。
4. 用户在支付宝的支付页完成支付后，支付宝会帮你把前端重定向到事先设置的return_url处，并同时向你的后端notify_url发送一条确认信息，如果没有支付，则不会执行以上的内容。

# 准备工作

我们首先需要获取支付宝的一些开发参数，如果想接入支付宝的话，就要准备一些材料，例如营业执照等，向支付宝提出申请，这部分内容可以通过 [首页 - 商家平台](https://b.alipay.com/page/portal/home) 查看，个人应该是申请不到这个能力的，这和微信支付的逻辑不谋而合，因此一些个人开发者做的小程序和网页，都是靠广告实现变现。

因为支付宝提供了沙箱环境，所以我们开发时，可以先使用沙箱进行模拟支付，等到支付宝的流程走完之后，只需要修改一下开发参数，而不需要修改任何的业务代码了。

获取沙箱账号和app，我们可以进入以下地址：[沙箱应用 - 开放平台](https://open.alipay.com/develop/sandbox/app) 

扫码登录，我们就可以看到

# 后端开发

SDK的作用是，将一些复杂业务封装起来，我们使用的时候就只需要调用就行了，而不需要重复造轮子。支付宝官方提供了一些语言的SDK，网址如下：[概述 - 支付宝文档中心](https://opendocs.alipay.com/open/54/103419) 

但是我们使用的是Go，所以只能去找一些非官方的SDK，并且对着文档去测试了，我们使用的是以下这个包：

```go
go get github.com/smartwalle/alipay/v3
```

首先，我们全局实例化一个支付客户端：

```go
var AlipayClient *alipay.Client

func initAlipay() {
	appId := "APPID"
	privateKey := "应用私钥"
	alipayPublicKey := "支付宝公钥"
	isProduction := false
	alipayClient, err := alipay.New(appId, privateKey, isProduction)
	if err != nil {
		panic(err)
	}
	err = alipayClient.LoadAliPayPublicKey(alipayPublicKey)
	if err != nil {
		panic(err)
	}
	AlipayClient = alipayClient
}
```

这部分的内容是初始化支付参数，在调起订单或者验签时，我们就不需要重复初始化了，接下来，当用户点击支付时，我们调用这个客户端的函数来生成url：

```go
func Pay(OutTradeNo string,Subject string,Body string,ClientType string,totalAmount float64) (response string, err error) {
	if !Check(totalAmount) {
		panic("金额错误！")
	}
    // TODO
    // insertToSql()
	var p = alipay.TradeWapPay{}
	p.NotifyURL = "异步通知地址"
	p.ReturnURL = "同步通知地址"
	p.Subject = Subject
	totalAmountStr := strconv.FormatFloat(totalAmount, 'f', 2, 64)
	p.TotalAmount = totalAmountStr
	p.OutTradeNo = params.OutTradeNo
	p.ProductCode = "QUICK_WAP_WAY"
	url, err := config.AlipayClient.TradeWapPay(p)
	if err != nil {
		return
	}
	return url.String(), nil
}
```

以上是生成支付链接的代码，我们需要在同时，将这个订单的内容插入到我们的数据库，以便之后的查询和更新。

我们可以在handler层解析参数，并把参数传到我们的业务函数中，实现MVC设计模式。

这段函数看起来就像是把参数都挂到我们生成的支付对象p上，非常的简单，有以下的一些要点：

1. 不要信任前端传来的金额参数，因为可以人为的修改，所以我们自行实现Check函数，确保金额没有被修改。
2. 同步通知地址的意思是，当我支付完成后，前端会重定向到哪里。假设我刚在购物车买了一件商品，那付完钱我就应该回到购物车，而不是一个我没见过的商品页。
3. 异步通知地址的意思是，支付宝向哪里发送支付成功消息，这个地址必须是https，并且不能有特殊符号”#“之类，也不能是内网地址。
4. 金额的单位是”元“，所以我们要使用两位小数，这里与微信支付有一些区别，微信支付使用的是int型，单位是”分“。
5. ProductCode字符串表示调起支付的类型，QUICK_WAP_WAY表示手机网站支付，会给你一个竖版h5的页面。

这样我们前端拿到地址后，使用window.location.href跳转就可以了。

验证签名也十分的简单，只需要调用函数就行了：

```go
func Notify(params string) {
	err := config.AlipayClient.VerifySign(params)
	if err !=nil {
		c.String(http.StatusOK, "fail")
		return
	}
	c.String(http.StatusOK, "success")
}
```

验证签名时，如果想自己实现，则需要调用rsa的验签方法，这部分支付宝给出了教程： [自行实现验签 - 支付宝文档中心](https://opendocs.alipay.com/common/02mse7) ，但是我们直接调用VerifySign函数，就省下了重复造轮子的繁琐。

验完签名后，需要向支付宝发送“success”这一个字符串，不需要包装，如果不发送或者发送fail，支付宝会尝试每隔一段时间，再向你的服务器发送一次，这个时间间隔是：4m,10m,10m,1h,2h,6h,15h，也就是说，25h之内会发送8次通知。

前端通知函数和后端类似，不同的是签名是前端传入的，验签通过后就不必向支付宝发送什么消息了，比较可靠的是异步通知，据说准确率非常高，几乎是可以认为100%的程度。

Notify之后的事情就好办了，我们可以调用下发奖励函数，给相应的用户，比如我从params中拿到了订单号，我就从数据库中，根据订单号找到这个用户，给他下发对应的奖励。因为前端也有同步通知，所以需要先做一层判断，判断奖励是否已经下发，防止重复下发。根据实际生产上的经验，异步通知往往会比同步通知先到达，所以，一定要在同步通知函数中，也做一层相同的判断。

至此，支付宝支付所有的逻辑就结束了。
